Stage 1:Webpages, Documents, and Structure

Web 101

The Web
The web works using the following elements html, the internet, http and server. Essential it is a vast number of computers in a network talking to each other. When a user connects to a website they make a request to a server via http, Hyper Text Transfer Protocol. The server searches for relevant HTML to send back to the user's computer. This is then displayed on the users screen via a web browser which can interpret the data.

HTML & Code

HTML Basics
Here is a great starting point for HTML resources and details on the vocabulary. Tags are always contained within angled brackets. Most tags have an opening tag and a closing tag. Void tags do not require a closing tag. Elements within html refer to everything within a set of opening and closing tags. HTML elements can be inline or block. Block essential means there is an invisible box around it. A paragraph tag is an example of a block element. Attributes are properties of html elements, eg the href attribute of an anchor tag to the URL for google

The Code
These are the instructions a programmer creates to enable a computer to run programmes and tasks in a syntax it understands.Computers can be said to be stupid as they need to follow this syntax exactly. There can be no general interpretations of what the computer thinks the programmer may mean. The computer is very unforgiving to any mistakes the programmer may make in their code. Programmers will struggle to remember all the details of a given code language variant but this is fine as there is a wealth of resources online.

The Tools and Adding Structure

Developer Tools
Some browsers such as Google chrome and Mozilla firefox have developer tools built in. These allow the users to inspect webpages and individual elements within them to understand how they are structured.

DOM
DOM stands for Document Object Model. It details how a website is structured, like an organisation chart within a company. It shows the relationship elements within an html document have with each other. Elements can be within elements.

Boxes boxes everywhere
The elements contained within an html document and the tree like structure are shown as boxes in a browser. Even if they dont appear to be box-like shapes to the naked eye, trust me they are. This is key to the structuring of a webpage.

Style it up!

CSS
CSS, cascading style sheet, is code written to control the style of HTML elements. Using CSS can save a considerable amount of time by avoiding repetition in grouping styles together. There can be multiple conflicting style rules within CSS effecting the same elements, however the most specific/granual level will always have priority.

There are three methods to styling:

  • Write style inline with your HTML. This is poor form!
  • Write CSS in the < head > of your HTML. This is ok for small projects
  • Link your HTML to a separate CSS file. This is best practise!
Quickfire: Creating a webpage

When creating a website follow these simple rules

  • Visually boxify
  • Look for repeated styles and semantics
  • Write HTML
  • Apply styles biggest to smallest
  • Work out how big the boxes should be
  • Then where to position boxes on the page
